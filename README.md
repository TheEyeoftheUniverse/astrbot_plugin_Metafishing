## ✨ 功能特点

- **完整的钓鱼主循环**：基础冷却 180s（鱼饵 5★ 起每星减 10% CD，10★ 上限 -60%），70% 基础成功率，装备/鱼饵/Buff 多源叠加
- **品质系统**：每条鱼有概率被判定为 ✨ 高品质，出售按 2 倍价值结算；触发概率经对数压缩，封顶 35%
- **多区域钓鱼**：默认 3 档（新手港湾 / 深海峡谷 / 传说之海），各区域独立稀有度分布、钓鱼成本、稀有鱼日配额、开放时段、通行证门槛、区域限定鱼
- **通行证驻留机制**：消耗型通行证开通 24 小时驻留续票，永久型通行证常驻校验，到期自动转移到初始区域
- **背包与库存**：鱼塘 480 → 999 → 9999 → 99999 → 999999 五档升级，水族箱独立容量，按稀有度批量转移
- **装备成长链**：鱼竿 / 饰品 / 鱼饵分别贡献品质 / 数量 / 稀有 / 价值四种加成，支持精炼、锁定保护、耐久度损坏卸下
- **道具与 Buff 系统**：钱袋开包、擦弹次数 +1、擦弹预言、稀有鱼加成、精炼保护盾、CD 重置、暗影斗篷、破灵符等 11 类即时/Buff 效果
- **多商店系统**：每个商店有独立时间窗、每日时间窗、限购上限，支持金币 / 钻石 / 道具 / 鱼等多种货币与多种奖励
- **玩家市场**：支持鱼、鱼竿、饰品、道具、大宗商品五类上架，5% 上架税率，匿名上架支持
- **期货系统**：三种大宗商品（鱼干 / 鱼卵 / 鱼油），市场情绪（恐慌→狂热五档）+ 突发事件，每日 3 次定时调价，持仓容量五档升级
- **抽卡与擦弹**：加权抽卡 / 单抽 / 十连 / 限时卡池 / 每日免费池；擦弹倍率分七档（末吉 1×→ 極大吉最高 1500×），含每日奖池累积
- **图鉴奖励**：鱼类图鉴按 `5% → 100%` 共 11 个节点发放一次性高级货币，总额 `6666` 钻石，支持命令端和玩家 WebUI 直接领取
- **社交对抗**：偷鱼（4h CD）+ 电鱼（2h CD，60% 基础成功率）+ 驱灵；转账手续费、累进税、排行榜、图鉴、图鉴奖励、称号、成就
- **科考系统**：探险（24h）/ 征服（48h）/ 圣域（72h）三档，队长 + 队员双许可证，五星目标鱼 + 卖鱼推进进度，邀请码加入，各自领奖
- **后台双 WebUI**：玩家端默认 `8888`，管理端默认 `7777`
- **Unity 安卓客户端**：基于 Unity 引擎的玩家移动端 App，通过 WebUI API 接入

## 🔧 安装方法

1. 将本插件放入 `data/plugins/` 目录，或通过 AstrBot 插件安装方式从仓库地址安装。
2. 重启 AstrBot 或重新加载插件。
3. 插件启动后会自动执行数据库迁移。
4. 如需管理员后台，请先配置 `webui.secret_key`。
5. 如需 Linux.do OAuth，请补全 `webui.oauth.linuxdo` 配置。

### 💼 期货系统

**机制说明**：期货市场有三种大宗商品——**鱼干**（低风险低收益，初始价 6000，波动 ±8%）、**鱼卵**（高风险高收益，初始价 12000，波动 ±12%）、**鱼油**（投机品，初始价 10000，波动 ±10%，事件驱动）。每日 9:00 / 15:00 / 21:00 三个时点自动调价，调价时按**市场情绪**（恐慌 10% / 悲观 20% / 中性 40% / 乐观 20% / 狂热 10%）+ 10% 概率的突发事件叠加波动率，单次涨跌幅封顶 20%。开通账户后获得初始 1000 持仓容量，按 10w / 50w / 200w / 1000w 金币阶梯升级到 2 / 5 / 10 / 20 倍。每笔交易收 5% 税；30 分钟内同方向重复交易自动合并均价。`/期货 历史` 看价格走势，`/期货 分析` 看 K 线和情绪指标。

### 🎰 抽卡与擦弹

**机制说明**：**抽卡**采用加权随机模型，物品在卡池里有不同权重；可单抽或十连，部分卡池为限时活动池，每日还有一个免费池供白嫖。所有抽卡结果记入 `/抽卡记录`，可作为对账凭证。**擦弹**是赌博类玩法：每日 3 次（可被擦弹次数+1 道具增加），投入金币换取**七档随机倍率**——末吉（1–2×）、小吉（2–3×）、吉（3–6×）、中吉（6–15×）、大吉（15–50×）、超大吉（50–200×）、極大吉（200–1500×）。**擦弹预言**道具可在投入前先看到本次倍率（精确预言）或档位（粗略预言）。失败投入会进入**每日奖池**，由触发奖池条件的玩家瓜分；服务器有"压制状态"机制，防止单日超大吉刷屏。

### 👥 社交与对抗

### 🔬 科考系统

**机制说明**：科考是**多人协作**玩法。三档时长：**探险**（24 小时）/ **征服**（48 小时）/ **圣域**（72 小时），队长发起需消耗对应许可证（探险证 / 征服证 / 圣域证），队员加入需消耗对应加入许可证（探险加入证 / 征服加入证 / 圣域加入证）。系统从鱼库随机选 5 个目标——1–3 星各 100/500/1000 条（按时长档），4 星 50/100/500，5 星 10/50/100。队员通过 `/出售` 鱼向团队进度贡献，到时自动结算并按贡献比例分配奖金，每个队员**各自领奖**（不会因离队丢失）。`/加入科考` 用邀请码进入队伍，过程中支持**特殊事件**触发，结算后历史保留 1 期供查询。

## 🌐 WebUI

### 玩家 WebUI

- 默认端口：`8888`
- 当前页面：首页总览、图鉴（含图鉴奖励领取）、背包、鱼塘、水族箱、市场、商店、期货、抽卡、酒馆、钓鱼区域
- 同时承担 **Unity 安卓客户端**的 HTTP API 后端
- 登录方式：
  - 游戏内账号密码
  - Linux.do OAuth

### 管理员 WebUI

- 默认端口：`7777`
- 当前管理能力：鱼类模板、鱼竿/鱼饵/饰品模板、抽卡池、玩家与库存、市场、商店、区域、称号、CSV 导入导出

## ⚙️ 关键配置

```json
{
  "webui": {
    "secret_key": "SecretKey",
    "port": 7777,
    "player_port": 8888,
    "oauth": {
      "linuxdo": {
        "enabled": false,
        "client_id": "",
        "client_secret": "",
        "redirect_uri": "http://localhost:8888/player/oauth/linuxdo/callback",
        "scope": "read",
        "user_id_field": "id",
        "nickname_field": "username",
        "auto_register": true,
        "allow_password_fallback": true,
        "proxy_url": ""
      }
    }
  },
  "fishing": {
    "cooldown_seconds": 180,
    "quality_bonus_max_chance": 0.35,
    "auto_bucket_count": 4
  },
  "steal": {
    "cooldown_seconds": 14400
  },
  "electric_fish": {
    "enabled": true,
    "cooldown_seconds": 7200,
    "base_success_rate": 0.6,
    "failure_penalty_max_rate": 0.5
  },
  "game": {
    "daily_reset_hour": 0,
    "wipe_bomb_attempts": 3
  },
  "user": {
    "initial_coins": 200
  }
}
```

> `fishing.auto_bucket_count` 控制后台自动钓鱼线程的分桶数（默认 4）：将玩家按 user_id 哈希分到 N 个桶里，每个 tick 只处理 1/N 玩家，单核瞬时 CPU 与 SQLite 写锁竞争降到 1/N，每个玩家被检查的总频率不变。设为 `1` 即关闭分桶，回退到原始全员单轮处理。建议 4–8。
