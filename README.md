## 项目现状

当前单一基线迁移会直接创建最新结构并写入最新静态数据，内置内容规模如下：

- `672` 条鱼
- `8` 个钓鱼区域
- `42` 根鱼竿
- `42` 个饰品
- `22` 种鱼饵
- `54` 种通用道具
- `15` 个期货商品
- `8` 个卡池
- `8` 家商店

数据库基线位于 [001_initial_setup.py](/mnt/d/Fishing/astrbot_plugin_fishing/core/database/migrations/001_initial_setup.py:1)，只保留一份最新初始化逻辑，不再依赖历史补丁迁移链。

## 功能概览

- 完整钓鱼主循环：注册、冷却、成功率、品质鱼、鱼塘容量、自动钓鱼、签到、钓鱼日志。
- 多区域成长线：当前为 `8` 个区域，支持区域成本、稀有度分布、开放时间、通行证校验、日稀有配额、到期自动迁回。
- 装备成长系统：鱼竿、饰品、鱼饵三套模板；耐久、精炼、锁定、在用状态、自动鱼饵启停。
- 背包与收藏：鱼塘、水族箱、按稀有度批量转入转出、水族箱容量升级。
- 商店与市场：系统商店支持多重成本和多重奖励；玩家市场支持鱼、鱼竿、饰品、道具、期货持仓上架。
- 交易所系统：`15` 个商品完全从数据库读取商品定义、初始价格和调价规则；支持开户、持仓扩容、自动到期卖出、历史走势、分析报告。
- 抽卡系统：`8` 个卡池，支持金币池/高级货币池、单抽、十连、多次十连、限时池、保底兑换物发放、称号奖励直授。
- 社交对抗：偷鱼、电鱼、转账、排行榜、税收记录、称号、成就、图鉴奖励。
- 科考系统：探险 / 征服 / 圣域三档，多人协作，卖鱼推进进度，奖励改为成员各自领取。
- 玩家 WebUI：背包、鱼塘、水族箱、市场、商店、交易所、抽卡、图鉴、钓鱼区域、酒馆。
- 管理 WebUI：用户、区域、鱼类、装备、道具、称号、商店、市场、交易所、卡池管理。

## 安装方法

1. 将插件放入 AstrBot 的 `data/plugins/` 目录，或通过仓库地址安装。
2. 重启 AstrBot 或重新加载插件。
3. 插件启动时会执行唯一基线迁移，生成最新 `fish.db`。
4. 若需要管理员后台，先配置 `webui.secret_key`。
5. 若需要玩家端第三方登录，补全 `webui.oauth.linuxdo`。

## 玩法说明

### 1. 钓鱼与区域

- `/钓鱼` 执行一次完整钓鱼流程，读取当前区域成本、时间窗和通行证状态。
- `/自动钓鱼` 切换后台代钓。自动钓鱼支持分桶轮询，减轻单轮 CPU 和 SQLite 写锁压力。
- `/钓鱼区域 [ID]` 查看或切换区域。当前内置 `8` 区，区域 2 以后基本都带有门票要求。
- 区域支持永久票、消耗票、驻留续票和失效后自动迁回初始区。

### 2. 背包、鱼塘、水族箱

- 鱼塘负责日常存鱼，容量可升级。
- 水族箱是独立收藏空间，可按单鱼或按稀有度批量转入转出。
- 玩家图鉴统计不依赖当前库存留存，而依赖已解锁鱼种聚合数据。
- 水族箱被酒馆抽中为“今日展览”时，其他玩家可以在酒馆留言；展览者可在自己的水族箱页管理展览评论。

### 3. 装备、鱼饵、道具

- 鱼竿和饰品支持精炼、锁定、出售、在用状态。
- 鱼竿带耐久度，损坏后会自动卸下。
- 鱼饵支持查看、使用，以及 `/启用自动鱼饵 [ID]`、`/停用自动鱼饵 [ID]` 控制是否参与自动钓鱼。
- 通用道具包含钱袋、区域门票、科考许可证、擦弹相关道具、称号授予道具等。
- 部分称号道具使用后会直接授予对应称号。

### 4. 商店与玩家市场

- 商店系统基于 `shops + shop_items + shop_item_costs + shop_item_rewards` 四张静态表。
- 每个商品可以配置多重消耗和多重奖励，不限于金币。
- 玩家市场支持上架：
  - 鱼
  - 鱼竿
  - 饰品
  - 道具
  - 期货持仓
- 市场支持匿名上架，装备展示精炼等级，商品会收上架税。

### 5. 交易所

- 当前不是旧版“3 商品写死方案”，而是数据库驱动的 `15` 商品交易所。
- 商品定义来自 `commodities` 表。
- 初始价格来自 `exchange_prices` 表。
- 调价规则来自 `exchange_commodity_rules` 表。
- 默认每日调价时点为 `9:00 / 15:00 / 21:00`，可通过配置调整。
- 交易所支持：
  - 免费开户
  - 持仓容量升级
  - 买入 / 卖出
  - 到期自动卖出并结算
  - 价格历史
  - 市场分析
- 玩家端 WebUI 会展示价格走势、持仓盈亏、市场情绪、供需状态。

### 6. 抽卡

- 当前内置 `8` 个卡池。
- 同时支持金币池与高级货币池。
- 支持单抽、十连、多次十连、限时卡池。
- 抽卡奖品支持鱼竿、饰品、鱼饵、通用道具、金币、称号。
- 部分卡池会额外按抽数发放保底兑换物。

### 7. 社交、排行与图鉴

- `/偷鱼 [@用户]`：从目标鱼塘随机偷取鱼。
- `/电鱼 [@用户]`：高风险高收益的批量掠夺玩法。
- `/排行榜`：查看金币榜、钓鱼次数榜、历史最高金币相关排行。
- `/鱼类图鉴`：查看图鉴完成情况。
- `/图鉴奖励`：统一查询并领取当前所有可领图鉴奖励。
- `/查看称号`、`/使用称号 [ID]`：称号查看和装备。
- `/查看成就`：查看成就进度。
- `/税收记录`：查看税收扣费历史。

### 8. 科考

- `/发起科考 [类型]`：发起三档科考之一。
- `/加入科考 [邀请码]`：使用邀请码加入队伍。
- `/科考状态`：查看当前进度。
- `/结束科考`：查看当前状态或完成说明，不再直接整队结算。
- 奖励现在按成员个人领取，不再依赖“队长统一结算发放”。
- 卖鱼行为会自动推进对应目标鱼的科考进度。

### 9. 酒馆与 WebUI 玩法

酒馆页不是单纯聊天页，当前已承载这些玩法：

- 公告栏
- 留言板
- 三类排行榜
- 每日擦弹公示
- 今日水族箱展览
- 进行中的科考公告板
- 科考奖励领取入口

其中“每日擦弹”支持奖池、公示榜、剩余次数展示和结果文本回显，属于 WebUI 已落地玩法。

## 关键配置

最常用的配置项如下，完整定义见 [_conf_schema.json](/mnt/d/Fishing/astrbot_plugin_fishing/_conf_schema.json:1)。

```json
{
  "webui": {
    "secret_key": "SecretKey",
    "port": 7777,
    "player_port": 8888
  },
  "tax": {
    "is_tax": true,
    "threshold": 1000000,
    "transfer_tax_rate": 0.05
  },
  "fishing": {
    "cooldown_seconds": 180,
    "quality_bonus_max_chance": 0.35,
    "auto_bucket_count": 4
  },
  "steal": {
    "cooldown_seconds": 14400
  },
  "electric_fish": {
    "enabled": true,
    "cooldown_seconds": 7200,
    "base_success_rate": 0.6,
    "failure_penalty_max_rate": 0.5
  },
  "game": {
    "daily_reset_hour": 0,
    "wipe_bomb_attempts": 3
  },
  "user": {
    "initial_coins": 200
  },
  "market": {
    "listing_tax_rate": 0.05
  },
  "exchange": {
    "update_timing": "9:00, 15:00, 21:00",
    "capacity": 1000,
    "tax_rate": 0.05,
    "merge_window_minutes": 30
  }
}
```

说明：

- `fishing.auto_bucket_count` 控制自动钓鱼分桶数，推荐 `4-8`。
- `exchange` 仍保留调度、税率、容量等运行配置。
- 交易所的商品列表、初始价格和调价规则不再依赖配置兜底，而是直接读取数据库静态表。
