# 执行计划：钓鱼概率与收益判定重构

## 内部执行等级

M：核心逻辑集中在 `FishingService` 与鱼模板抽样工具，单链路后端重构。

## 步骤

1. 冻结本次需求文档，限定只动后端概率/判定逻辑。
2. 重写同星级鱼抽样：
   - 以 `base_value` 排序分三档价值池。
   - 默认按 `20/60/20` 抽池。
   - 价值加成只把中档概率搬到高档，低档固定保留。
3. 收敛稀有度加成：
   - 继续从 `1-3 星` 向 `4-5 星` 搬运概率。
   - `6+ 星` 完全独立。
   - 对叠加后的实际转移比例做温和压缩，降低 5 星膨胀。
4. 将品质/数量改为加法累计：
   - 各来源先转换为相对 `1.0` 的增量后累加。
   - 按固定星级衰减系数判定。
   - 数量最多额外 `+1`，`6+ 星` 永不增产。
5. 做静态验证，确认编译通过并覆盖关键验收点。

## 验证命令

- `python3 -m py_compile core/services/fishing_service.py core/utils.py`
- 静态脚本验证：
  - 同星级高价值加成下低价值池仍有稳定概率。
  - `6+ 星` 数量衰减为 `0.0`，永远只出单条。
  - 4 星、5 星、6+ 星的品质/数量概率低于 1-3 星。
