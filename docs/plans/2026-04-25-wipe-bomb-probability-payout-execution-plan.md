# 执行计划：擦弹概率与奖池结算调整

## 内部执行等级

M：单服务核心逻辑调整，影响面集中在 `GameMechanicsService`。

## 步骤

1. 将擦弹普通/抑制模式权重表收敛为单一类级配置，供预测和实际擦弹共用。
2. 重设倍率区间：
   - 亏损区间总权重约 50%。
   - 持平区间为固定 `1.0x`。
   - 小赚从 `1.01x` 开始。
3. 调整奖池结算：
   - `<=2.0x` 盈利直接兑现。
   - `>2.0x` 的额外收益从奖池支付。
   - 亏损继续注入奖池。
4. 增加最小盈利处理，避免 `int()` 截断导致 `>1.0x` 结果实际持平。
5. 用本地 Python 静态计算验证权重、期望倍率和典型结算场景。

## 验证命令

- `python3 -m py_compile core/services/game_mechanics_service.py`
- 静态概率计算脚本：验证普通模式亏损率、持平率、盈利率和期望倍率。
- 静态结算脚本：验证 `1.5x` 不走奖池，`5x` 只让 `2x` 以上额外收益走奖池。
