# 钓鱼区域通行证驻留续票执行计划

## Internal Grade

L：单服务核心逻辑变更，涉及数据库迁移、仓储方法、服务层校验和编译验证；无需并行子代理。

## 执行批次

1. 数据结构
   - 新增迁移 `043_add_user_zone_stays.py`。
   - 增加 `user_zone_stays` 表，按 `user_id + zone_id` 保存当前消耗型驻留状态。

2. 仓储接口
   - 在库存仓储中增加读取、写入、删除、到期查询驻留记录的方法。
   - 解析/写入 ISO 时间字符串，保持与现有区域时间处理风格一致。

3. 服务语义
   - 将通行证检查拆成永久票分支和消耗票驻留分支。
   - 进入消耗票区域时扣票并写入 24 小时驻留。
   - 主动切出区域时清理旧区域驻留。
   - 到期检查时续票或回传。
   - 钓鱼前校验当前区域驻留有效性。

4. 后台检查
   - 从每日刷新分支中分离通行证检查。
   - 自动钓鱼线程周期性执行到期驻留检查，不再依赖每日刷新点。

5. 验证
   - `python3 -m py_compile` 覆盖变更文件。
   - 使用轻量内存/临时库 smoke 检查仓储迁移和服务关键分支，若环境限制则记录。

## 回滚规则

- 代码回滚时同时移除迁移使用点；数据库迁移已创建的表可保留为空表，不影响旧逻辑。
- 不修改用户已有 `README.md` 变更。

## 完成语言规则

只有在编译验证完成后，才能声明代码修改完成；若 smoke 测试受环境限制，需明确说明。
