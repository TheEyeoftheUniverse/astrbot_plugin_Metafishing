# 需求冻结：签到奖励与刷新口径重构

## 目标

重构 `astrbot_plugin_fishing` 的签到系统，使签到成为高级货币的主要稳定来源，并移除每日免费补给池带来的重复日常奖励链路。同时统一签到与其他日常系统的刷新口径，降低玩家对“今天是否可签”的理解成本。

## 范围

- 调整签到奖励计算规则。
- 移除签到自动抽取每日免费池的逻辑。
- 统一签到判定与连签判定的每日刷新口径。
- 保留现有签到入口、WebUI 签到入口和用户连续签到字段。
- 不重做科考系统。
- 不新增后台配置页面。
- 不新增签到日历、补签、月卡、赛季通行证等扩展玩法。

## 冻结规则

### 1. 签到奖励

- 签到基础倍率由当前连续签到天数决定。
- 当天签到成功后，先更新连续签到天数，再按更新后的连续签到天数计算奖励。
- 金币奖励为：
  - `random(100, 300) * consecutive_login_days`
- 高级货币奖励固定为：
  - `1 * consecutive_login_days`
- 金币基础值继续使用 `100-300` 随机区间。
- 不使用旧的 `signin.min_reward`、`signin.max_reward` 配置覆盖该随机区间作为默认奖励来源。
- 不使用旧的 `signin.consecutive_bonuses` 节点额外奖励作为默认奖励来源。

### 2. 连签规则

- 如果玩家在上一刷新周期内完成过签到，则本次签到延续连签。
- 如果玩家在上一刷新周期内没有完成签到，则本次签到前将连续签到天数重置为 `0`。
- 本次签到成功后，连续签到天数增加 `1`。
- 连续签到奖励不封顶。
- 不因为单赛季长度、自然月切换或累计天数过高而截断奖励。

### 3. 刷新口径

- 签到系统使用全局 `game.daily_reset_hour` 作为每日刷新点。
- “今日是否已签到”不再按自然日 `00:00` 判断。
- “昨天是否签到”不再按自然日前一天判断，而是按上一个刷新周期判断。
- 刷新周期与稀有鱼配额、税收、擦弹等日常系统保持同一口径。

### 4. 每日免费池

- 签到不再自动抽取每日免费池。
- 签到成功消息中不再包含每日补给、免费抽卡、补给领取失败等内容。
- 签到奖励只包含金币、高级货币和连续签到信息。
- 是否保留后台中已有的免费卡池管理能力不属于本次需求范围。
- 是否删除数据库中已存在的免费卡池数据不属于本次需求范围。

### 5. 高级货币定位

- 签到是高级货币的主要稳定来源。
- 科学考察继续作为高级货币的额外来源。
- 接受长期连续签到玩家获得明显更高的日常高级货币收益。
- 不为防止长期连签增长而设置奖励上限。

## 验收标准

- 连续签到第 `1` 天奖励为 `100-300` 金币和 `1` 高级货币。
- 连续签到第 `30` 天当天奖励为 `3000-9000` 金币和 `30` 高级货币。
- 连续签到 `30` 天累计签到金币期望约为 `93000`，实际受每日随机基础金币影响；累计高级货币为 `465`。
- 签到成功后不会触发任何免费抽卡逻辑。
- 没有免费池、免费池为空或免费池已领取，都不影响签到成功。
- 修改 `game.daily_reset_hour` 后，签到可用状态和连签延续状态按新的刷新点判断。
- 在同一刷新周期内重复签到会被拒绝。
- 跨过一个完整刷新周期未签到后，再次签到会从连续签到第 `1` 天重新开始。
- QQ 命令签到、玩家 WebUI 签到、用户端 API 签到使用同一套奖励与刷新判定。

## 非目标

- 不设计签到补签机制。
- 不设计每周、每月或赛季签到宝箱。
- 不调整科学考察高级货币产出。
- 不调整高级货币卡池价格。
- 不调整商店、水族箱、市场等高级货币消费端。
- 不处理历史签到记录的数据迁移。
