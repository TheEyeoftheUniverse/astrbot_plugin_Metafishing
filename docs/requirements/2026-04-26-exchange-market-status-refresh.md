# 交易所市场状态动态刷新

日期：2026-04-26

## 目标

让交易所页面中的以下字段不再是固定占位值，而是随每日价格刷新和持仓状态动态变化：

- 市场情绪
- 价格趋势
- 供需状态

## 当前问题

`ExchangePriceService.get_market_status()` 当前对上述三个字段返回硬编码常量：

- `market_sentiment = "neutral"`
- `price_trend = "stable"`
- `supply_demand = "平衡"`

因此即使价格历史正常更新，页面文案也永远不变。

## 期望行为

- 价格趋势：
  基于最近价格记录与上一周期价格记录的变化，给出“上涨 / 下跌 / 稳定”等描述。
- 市场情绪：
  基于整体涨跌方向和涨跌幅强弱，给出“乐观 / 悲观 / 中性”等描述。
- 供需状态：
  基于当前用户总持仓和持有人数分布，给出“供过于求 / 需求旺盛 / 基本平衡”等描述。

## 约束

- 不改动现有交易所价格历史表结构。
- 不依赖外部服务。
- 保持现有页面接口字段名不变。
- 当历史数据不足时，要返回合理的默认描述，而不是报错。

## 验收标准

- 交易所价格历史继续正常显示。
- 当价格发生明显上涨或下跌时，`price_trend` 与 `market_sentiment` 会发生对应变化。
- 当持仓量和持有人数变化时，`supply_demand` 会跟着变化。
- `python3 -m py_compile` 通过。
