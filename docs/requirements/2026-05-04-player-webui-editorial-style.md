# 玩家 WebUI Minimal Editorial Writing Workspace 风格重构需求

日期：2026-05-04

## 目标

将 `player/` 玩家端 WebUI 重构为 “minimal editorial writing workspace” 风格：暖纸背景、编辑感 serif 标题、细分隔线、少卡片、单个主 CTA，并保留明亮的游戏感。

## 交付物

- 更新玩家端全局布局与样式系统。
- 优化首页与登录页的首屏表达，使主操作更明确。
- 保持现有 Jinja 路由、表单、按钮事件、通知面板与业务逻辑不变。

## 约束

- 不改数据库、服务层和管理端 WebUI。
- 不覆盖当前未提交的 `player/templates/oauth_complete.html` 行为修复。
- 保持 Bootstrap 5 与 Font Awesome 依赖，不新增构建链。
- 样式应通过玩家端 CSS 与少量模板类名完成，避免大范围模板重写。

## 验收标准

- 页面背景呈暖纸质感，整体不是冷灰/蓝色后台风。
- 页面标题使用 serif 字体，带编辑/刊物式层级。
- 主要内容以细线、留白和轻量面板分隔，减少厚重阴影与彩色卡片头。
- 首页只有一个明确主 CTA：“开始钓鱼”；其他操作降级为次级/文本按钮。
- 游戏感通过高亮色、徽章、进度、资源数值和交互反馈保留，但不形成多色块堆叠。
- 模板语法能通过 Python 编译检查，CSS 文件存在且可被布局引用。

## 非目标

- 不做后端接口改造。
- 不引入 SPA、打包器或新 UI 框架。
- 不重构管理端 `manager/` 模板。
